# AI Prompt Engineering Lab

## Overview
This project demonstrates prompt engineering techniques applied to three real-world scenarios:
1. Retail Inventory Analysis
2. Website Analytics Debugging
3. Customer Segmentation via SQL

The goal was to explore how different prompting strategies affect AI-generated code quality, debugging, and optimization.

---

## Scenario 1: Retail Inventory Analysis
### Objective:
Analyze inventory turnover, identify slow-moving stock, and predict stockouts.

### Techniques Used:
- Data cleaning and validation
- Date-based calculations
- Visualization of stock trends

---

## Scenario 2: Website Analytics Debug
### Objective:
Fix incorrect engagement metrics such as bounce rate > 100% and invalid session times.

### Improvements:
- Corrected grouping logic
- Fixed time aggregation issues
- Added validation for session data

---

## Scenario 3: Customer Segmentation (SQL)
### Objective:
Segment users based on activity, spending, and preferences.

### Approach:
- Used CTEs for readability and optimization
- Applied percentile ranking for high-value customers
- Filtered active users (last 30 days)

---

## Tools Used
- Python (Pandas, Matplotlib)
- SQL
- Jupyter Notebook
- Prompt Engineering (AI-assisted coding)

---

## Key Learnings
- Prompt structure significantly impacts AI output quality
- Iterative refinement improves accuracy and efficiency
- Validation is essential in AI-generated analytics code
